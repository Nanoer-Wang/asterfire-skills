# 概览

这是一个符合 Asterfire 平台规范的最小 Kit 示例。它接收一个输入文件，在当前工作目录生成结果文件和中文 `report.md`。

# 功能

- 从 `kwargs['args']` 读取输入参数。
- 对输入文件进行存在性检查。
- 输出文件写入当前工作目录。
- 使用 `@tool_io(outputs={...})` 显式声明输出端口。
- 运行结束返回 dict，并保持输出 key 与返回 key 完全一致。
- 自动生成包含输入、输出、文件存在性和警告信息的中文 `report.md`。
- 提供 `demos/` 目录，可在前端一键加载示例输入。

# 输入

| 参数 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| input_file | file | 是 | 需要处理的输入文件 |

# 输出

| 输出 | 类型 | 说明 |
| --- | --- | --- |
| output_file | 文件 | 示例结果文件 |
| report | Markdown 文件 | 平台运行报告 |
| success | 布尔值 | 是否成功生成主要输出 |

# 注意事项

- 新建真实 Kit 时，请删除与任务无关的字段，只保留必要输入。
- `SIF` 必须使用真实镜像，不要保留占位值。
- 如果项目已有 `Makefile`，不要覆盖原文件。
- 修改 `input.json` 后，请同步更新 `demos/demos.json`，确保 Demo 字段和示例文件仍然可用。
