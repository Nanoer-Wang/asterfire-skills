# 概览

这是一个由普通项目适配而来的 Asterfire 平台 Kit。它通过平台表单接收输入，在平台运行环境中调用原项目逻辑，并在当前工作目录生成结果文件和中文 `report.md`。

# 功能

- 从 `kwargs['args']` 读取平台表单参数。
- 检查输入文件或参数是否有效。
- 调用原项目函数或命令行入口。
- 把输出写入当前工作目录。
- 返回平台可识别的输出端口。
- 生成中文运行报告，记录输入、命令、输出、stdout、stderr、警告和假设。
- 提供 `demos/` 目录，可在前端一键加载示例输入。

# 输入

请根据真实 `config/input.json` 补充输入说明。这里只保留用户运行该 Kit 必须填写或确实需要调节的参数。

# 输出

请根据 `@tool_io(outputs={...})` 和 `return {...}` 补充输出说明。

# 运行环境

- SIF：`{{SIF}}`
- CPU：`{{CPU}}`
- GPU：`{{GPU}}`
- NETWORK：`{{NETWORK}}`

# 注意事项

- 本 Kit 的 `SIF` 必须来自用户指定或 SIF 注册表，不允许使用占位值。
- 所有输出写入当前工作目录。
- 如果原项目依赖外部命令，该命令必须存在于 SIF 镜像中。

- 修改 `input.json` 后，请同步更新 `demos/demos.json`，确保 Demo 字段和示例文件仍然可用。
